# RFC-0012.10 - Misaka Final Desktop Stability, Cloud Voice & Command System

## Objetivo

Transformar a Misaka Desktop em uma versao estavel para uso diario no PC, com comandos digitados e por voz passando pelo mesmo roteador deterministico antes do LLM.

## Decisoes

- Cloud Voice e o modo principal.
- Web Speech vira fallback opcional.
- Native Daemon vira fallback opcional.
- O frontend nunca recebe API key.
- Audio nao e salvo por padrao.
- A bridge Electron nao executa shell arbitrario.
- A Misaka so anuncia sucesso depois do retorno real da acao.

## Backend

Novo modulo:

```text
backend/app/voice/
  errors.py
  routes.py
  schemas.py
  service.py
  providers/
    base.py
    mock.py
    openai_provider.py
```

Endpoints:

- `GET /api/voice/status`
- `POST /api/voice/transcribe`

Provider `mock` e testado. Provider `openai` fica configuravel por ambiente e retorna erro seguro quando `OPENAI_API_KEY` esta ausente.

## Dashboard

`dashboard/voiceWake.js` agora implementa `CloudVoiceRecorder` e mantem `VoiceWakeController` como alias compatilvel. A UI mostra:

- modo de entrada de voz;
- modo de comando;
- status;
- ultimo texto ouvido;
- comando detectado;
- erro atual;
- teste de microfone;
- teste de transcricao.

## Electron

- Logger em arquivo.
- Tratamento de `EPIPE`.
- Permissao de microfone/media permitida; demais permissoes negadas.
- Tray envia ativar/desativar escuta mesmo se a dashboard ainda estiver carregando.
- Build gera NSIS e portable.

## Testes

Executados:

- `node --check` para dashboard e desktop.
- `node dashboard/voiceWake.test.js`.
- `python -m pytest` com 324 testes.
- `npm install`.
- `npm start` com Electron vivo apos 12 segundos.
- `npm run dist`, gerando setup e portable.

Nao executados automaticamente:

- fala real por microfone;
- teste em Vivaldi;
- cliques manuais no tray;
- abertura real de todos os apps pela UI.

Esses itens estao no plano manual e nao foram marcados como PASS no quality gate.
