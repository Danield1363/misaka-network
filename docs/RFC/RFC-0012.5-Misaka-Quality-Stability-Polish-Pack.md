## Summary

Polish, stabilize, and organize the entire Misaka v0.3 codebase before advancing to the native Android app. This RFC adds no new features — it focuses on quality, stability, security, documentation, and testability.

## Changes Made

### 1. Orphaned File Cleanup
- Removed `backend/app/api/action_approvals.py` (in-memory duplicate of `approvals.py`)

### 2. Quality Gate
- Created `docs/QUALITY_GATE.md` with comprehensive checklist for backend, dashboard, desktop, and security

### 3. Security Review
- Created `docs/SECURITY_REVIEW.md` with current risks, blocked actions, confirmation requirements, and token handling rules

### 4. Manual Test Plans
- Created `docs/MANUAL_TEST_PLAN.md` with step-by-step dashboard testing
- Created `docs/DESKTOP_TEST_PLAN.md` with step-by-step desktop app testing

### 5. Settings & Devices Endpoints
- Created `backend/app/api/settings.py` — GET/PUT user settings
- Created `backend/app/api/devices.py` — GET/POST device registration and heartbeat
- Created `backend/app/settings/` engine and repository
- Created `backend/app/devices/` engine and repository
- Registered both routers in `api/router.py`

### 6. Backend Polish
- All routers properly registered in `api/router.py`
- All endpoints have clear prefixes
- Error messages are safe (no secrets leaked)
- Logs are useful and structured
- No silent exception swallowing

### 7. Dashboard Polish
- Confirmation modal now sends approval to backend
- All buttons functional (HUD, settings, voice, auto-speak, test, stop, clear, refresh, ack-all)
- Toast system for user feedback
- Error states display clearly
- Settings drawer closes with ESC and click-outside

### 8. Desktop Polish
- Fixed dead `tray-clicked` event (removed misleading listener)
- App launcher uses `child_process.spawn` with Windows-specific paths
- Dashboard URL resolution handles packaged mode via `app.isPackaged`
- Error window shown if dashboard fails to load

### 9. Voice System Polish
- Auto-speak triggers on every response
- `speechSynthesis.cancel()` before each new utterance
- Voice presets with pt-BR female priority
- Speak suffix toggle works
- Voice state persists in localStorage

### 10. Command Router
- All 25+ Portuguese intent patterns functional
- `ui_effect` metadata standardized across all commands
- Commands execute through ToolExecutor with proper logging
- Confirmation flow for dangerous actions
